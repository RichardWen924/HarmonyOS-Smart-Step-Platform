# HarmonyOS 真机调试配置指南

## 问题说明
在 HarmonyOS 真机调试时，无法直接访问 `localhost` 地址，因为真机中的 localhost 指向设备本身，而不是开发电脑。

## 解决方案

### 方案一：使用电脑 IP 地址（推荐）

#### 步骤 1：获取电脑 IP 地址
1. **Windows 系统：**
   - 打开命令提示符（cmd）
   - 输入 `ipconfig`
   - 查找 "IPv4 地址"，通常是 `192.168.x.x` 格式

2. **Mac/Linux 系统：**
   - 打开终端
   - 输入 `ifconfig` 或 `ip addr`
   - 查找 `en0` 或 `wlan0` 的 `inet` 地址

#### 步骤 2：修改应用配置
在 `AppConfig.ets` 中修改 `baseUrl` 配置：

```typescript
class AppConfigImpl {
  // API 基础地址 - 修改为你的电脑 IP 地址
  baseUrl: string = 'http://192.168.1.100:8080/api' // 替换为你的电脑 IP
  
  // 其他配置保持不变...
}
```

#### 步骤 3：确保后端服务可访问
- 确保后端服务在 8080 端口运行
- 检查防火墙设置，允许 8080 端口的入站连接
- 确保电脑和手机在同一 Wi-Fi 网络下

### 方案二：使用 ngrok 或类似工具（备用）

如果方案一不可行，可以使用内网穿透工具：

1. **安装 ngrok：**
   ```bash
   # 下载 ngrok
   ngrok http 8080
   ```

2. **获取公网地址：**
   - ngrok 会提供一个类似 `https://abc123.ngrok.io` 的地址

3. **修改配置：**
   ```typescript
   baseUrl: string = 'https://abc123.ngrok.io/api'
   ```

### 方案三：使用域名（生产环境）

对于生产环境，必须使用合法的域名：

```typescript
baseUrl: string = 'https://api.your-app.com/api'
```

## 配置说明

### AppConfig.ets 配置文件详解

```typescript
class AppConfigImpl {
  // API 基础地址 - 根据环境修改此项
  baseUrl: string = 'http://localhost:8080/api'
  
  // 请求超时时间（毫秒）
  timeout: number = 10000
  
  // 最大重试次数
  maxRetries: number = 3
  
  // 重试延迟（毫秒）
  retryDelay: number = 1000
  
  // Token 存储键名
  tokenKey: string = 'smart_step_token'
  
  // 用户信息存储键名
  userKey: string = 'smart_step_user'
  
  // 步数存储键名
  stepKey: string = 'smart_step_data'
  
  // 自动同步步数的时间（小时，24 小时制）
  autoSyncHour: number = 22
  
  // 每日步数目标
  dailyStepGoal: number = 10000
  
  // 积分兑换比例（步数：积分）
  exchangeRate: number = 100 // 100 步 = 1 积分
  
  // 每日最大兑换步数
  maxDailyExchange: number = 10000
  
  // 是否启用日志
  enableLog: boolean = true
  
  // 是否启用 Mock 数据（开发阶段）
  enableMock: boolean = true
  
  // 版本号
  version: string = '1.0.0'
  
  // 应用名称
  appName: string = '智能计步器'
  
  // 最小支持的版本号
  minVersion: string = '1.0.0'
}
```

## 网络权限配置

确保 `module.json5` 中已正确配置网络权限：

```json5
{
  "requestPermissions": [
    {
      "name": "ohos.permission.INTERNET",
      "reason": "用于与服务器同步步数和积分数据",
      "usedScene": {
        "abilities": ["EntryAbility"],
        "when": "always"
      }
    }
  ]
}
```

## 常见问题排查

### 问题 1：网络请求失败
- **检查项：**
  - 手机和电脑是否在同一网络
  - 防火墙是否阻止了 8080 端口
  - 后端服务是否正常运行
  - `baseUrl` 配置是否正确

### 问题 2：证书错误（HTTPS）
- **解决方案：**
  - 开发环境使用 HTTP
  - 生产环境使用合法的 SSL 证书

### 问题 3：跨域问题
- **解决方案：**
  - 后端配置 CORS 头信息
  - 或者使用代理服务器

## 最佳实践

1. **开发阶段：** 使用 IP 地址 + Mock 数据
2. **测试阶段：** 使用测试服务器 + 真实数据
3. **生产阶段：** 使用正式域名 + 真实数据
4. **日志管理：** 开发环境启用详细日志，生产环境禁用

## 配置验证

创建测试页面验证网络连接：

```typescript
// 网络连接测试组件
@Component
export struct NetworkTest {
  @State message: string = '点击测试连接'

  build() {
    Column() {
      Button(this.message)
        .onClick(() => {
          this.testConnection()
        })
    }
  }

  async testConnection() {
    try {
      this.message = '测试中...'
      const apiClient = ApiClient.getInstance()
      const result = await apiClient.getUserPoints()
      this.message = '连接成功！'
    } catch (error) {
      this.message = '连接失败：' + error.message
    }
  }
}
```

## 快速切换配置

### 开发环境配置
```typescript
baseUrl: string = 'http://192.168.1.100:8080/api'
enableMock: boolean = true
enableLog: boolean = true
```

### 生产环境配置
```typescript
baseUrl: string = 'https://api.your-app.com/api'
enableMock: boolean = false
enableLog: boolean = false
```

通过以上配置，您的 HarmonyOS 应用就可以顺利对接真实的后端接口了！

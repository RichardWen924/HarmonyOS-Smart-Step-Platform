## 🔍 连接问题诊断

### 可能的原因
1. **后端服务未运行**
   - 服务器 `192.168.59.102:8080` 可能未启动
   - 检查后端服务状态

2. **网络连接问题**
   - 设备无法访问该IP地址
   - 防火墙或网络限制

3. **端口访问限制**
   - 端口8080可能被防火墙阻止
   - 需要配置网络权限

4. **IP地址变化**
   - 后端服务器IP可能已更改
   - 需要更新API地址

## 🛠️ 解决方案

### 方案一：使用Mock模式（推荐）
```typescript
// 在 AppConfig.ets 中设置
enableMock: true  // 使用模拟数据
```

**优点**:
- 无需后端服务器
- 所有功能均可正常使用
- 适合开发和演示

### 方案二：配置真实后端

#### 步骤1：启动后端服务
确保后端服务正在运行：
```bash
# 启动后端服务（具体命令取决于您的后端技术栈）
npm start
# 或
java -jar your-app.jar
# 或
python app.py
```

#### 步骤2：更新API配置
修改 `AppConfig.ets`:
```typescript
baseUrl: 'http://your-server-ip:port/api'  // 替换为实际地址
enableMock: false                         // 切换到真实模式
```

#### 步骤3：测试连接
使用网络诊断工具验证连接：
1. 进入登录页面
2. 点击"测试后端连接"
3. 运行完整诊断

### 方案三：使用本地开发服务器

#### 配置本地服务器
```typescript
// 本地开发环境配置
baseUrl: 'http://localhost:8080/api'
// 或
baseUrl: 'http://10.0.2.2:8080/api'  // Android模拟器专用
```

#### 启动本地后端
```bash
# 假设使用Node.js后端
cd backend
npm install
npm run dev
```

## 🌐 后端服务配置示例

### Node.js Express后端
```javascript
// server.js
const express = require('express');
const app = express();
const PORT = 8080;

app.use(express.json());

// 健康检查端点
app.get('/api/health', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date() });
});

// 登录端点
app.post('/api/auth/login', (req, res) => {
  // 实现登录逻辑
});

app.listen(PORT, '0.0.0.0', () => {
  console.log(`Server running on port ${PORT}`);
});
```

### 启动命令
```bash
node server.js
```

## 🔄 模式切换指南

### Mock → 真实模式
1. 确保后端服务运行
2. 修改 `AppConfig.ets`:
   ```typescript
   enableMock: false
   baseUrl: 'http://your-actual-server/api'
   ```
3. 重新编译应用
4. 测试连接

### 真实 → Mock模式
1. 修改 `AppConfig.ets`:
   ```typescript
   enableMock: true
   ```
2. 重新编译应用

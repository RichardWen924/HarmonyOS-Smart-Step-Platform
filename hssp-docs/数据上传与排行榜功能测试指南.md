# 数据上传与排行榜功能测试指南

## 📋 功能说明

### 1. 数据上传功能
- **作用**：将前端步数数据同步到数据库和Redis
- **触发时机**：
  - 用户手动点击"上传数据"按钮
  - 用户查看排行榜时自动触发
- **更新内容**：
  - 数据库：`step_record`表（步数记录）、`user`表（总步数、剩余步数）
  - Redis：日榜、周榜、月榜的ZSet数据

### 2. 排行榜功能
- **数据源**：从数据库实时查询所有用户的步数记录
- **显示内容**：排名、昵称/用户名、头像、步数
- **周期类型**：日榜、周榜、月榜
- **自动上传**：进入排行榜页面时自动上传当前步数到Redis

```typescript
// 新增方法：上传当前步数到服务器（确保Redis中有最新数据）
private async uploadCurrentSteps(): Promise<void> {
  try {
    const currentSteps = this.stateStore.todaySteps
    console.log('RankingPage: 准备上传步数:', currentSteps)
    
    if (currentSteps > 0) {
      const response = await this.apiClient.uploadSteps(
        currentSteps,
        new Date().toISOString().split('T')[0]
      )
      
      if (response.code === 1) {
        console.log('RankingPage: 步数上传成功，已更新Redis')
      } else {
        console.warn('RankingPage: 步数上传失败:', response.message)
      }
    } else {
      console.log('RankingPage: 今日步数为0，跳过上传')
    }
  } catch (error) {
    console.error('RankingPage: 上传步数异常:', error)
    // 不阻断排行榜加载，仅记录错误
  }
}
```

**效果**: 
- ✅ 每次进入排行榜页面时，自动上传当前步数
- ✅ 确保Redis中的数据是最新的
- ✅ 即使上传失败也不影响排行榜显示

## ⚠️ 常见问题

### 问题1：排行榜只显示当前用户

**原因**：数据库中只有当前用户有步数记录

**解决方案**：
1. 执行测试数据SQL脚本创建多个测试用户
2. 或者邀请其他用户注册并上传步数

### 问题2：Redis数据与数据库不一致

**原因**：旧代码使用`incrementScore`导致重复累加

**解决方案**：
1. 清空Redis中的旧数据：
   ```bash
   redis-cli
   KEYS hssp:rank:*
   DEL hssp:rank:daily:20260410
   DEL hssp:rank:week:202615
   DEL hssp:rank:month:202604
   ```
2. 重新上传步数，让系统重新生成Redis数据

### 问题3："我的"页面数据不更新

**原因**：页面没有刷新机制

**解决方案**：
- 已添加`onPageShow()`方法，每次显示页面时自动刷新
- 如果仍有问题，检查控制台日志确认API调用是否成功

### 问题4：上传步数后Redis未更新

**检查点**：
1. 后端日志是否有"Redis日榜更新成功"的记录
2. Redis连接是否正常
3. 检查`application.yml`中的Redis配置是否正确

## 📊 预期结果

### 数据流

```
前端步数变化
    ↓
用户点击"上传数据" 或 进入排行榜页面
    ↓
调用 POST /user/steps/upload
    ↓
后端处理：
  1. 更新 step_record 表
  2. 更新 user 表（total_step, remaining_step）
  3. 更新 Redis（日榜、周榜、月榜）
    ↓
前端收到成功响应
    ↓
重新加载用户信息（如果是手动上传）
    ↓
UI 显示最新数据
```

### Redis数据结构

```
ZSET hssp:rank:daily:20260410
  member: userId (Long)
  score: steps (Double)
  
ZSET hssp:rank:week:202615
  member: userId (Long)
  score: weekTotalSteps (Double)
  
ZSET hssp:rank:month:202604
  member: userId (Long)
  score: monthTotalSteps (Double)
```

## 📝 注意事项

1. **Redis键命名规范**：
   - 日榜：`hssp:rank:daily:{yyyyMMdd}`
   - 周榜：`hssp:rank:week:{yyyy}{ww}`
   - 月榜：`hssp:rank:month:{yyyyMM}`

2. **步数上传策略**：
   - 前端上传的是**当天的总步数**，不是增量
   - 后端会比较新旧值，取较大值更新
   - Redis使用`add`方法覆盖，不是`incrementScore`累加

3. **性能优化建议**：
   - 如果用户量很大，可以考虑定时任务批量更新Redis
   - 周榜和月榜可以缓存计算结果，避免频繁查询数据库

4. **数据安全**：
   - 上传步数时需要有效的Token
   - 后端会验证用户身份
   - Redis操作失败不会影响主流程

# 步数同步问题诊断与修复方案

## 🔍 问题分析

### 1. 步数回退问题
**根本原因**：后端 `uploadSteps` 方法中的步数计算逻辑错误

**问题代码位置**：
`hssp-backend/hssp-service/hssp-user/src/main/java/com/hssp/service/service/impl/UserServiceImpl.java`

**具体问题**：
```java
// 错误的逻辑：
if (recordDate.equals(LocalDate.now())) {
    // 获取今天之前的步数（用于计算总步数的增量）
    Integer previousTodaySteps = existingRecord != null ? existingRecord.getStepCount() : 0;
    int stepDiff = todaySteps - previousTodaySteps;
    
    // 更新用户的今日步数（remaining_step）：直接设置为今天的步数
    user.setRemainingStep(todaySteps);
    
    // 更新总步数：累加差值（如果差值为负数，说明步数减少了，也要相应减少总步数）
    Integer oldTotalStep = user.getTotalStep() != null ? user.getTotalStep() : 0;
    int newTotalStep = oldTotalStep + stepDiff;
    user.setTotalStep(newTotalStep);
}
```

**问题分析**：
- 当步数减少时（如设备重启），`stepDiff` 为负数
- 这会导致总步数减少，造成数据回退
- 正确的逻辑应该是总步数只增不减

### 2. 同步失败问题
**原因**：Token验证和网络请求处理逻辑不完善

**问题位置**：
- `entry/src/main/ets/services/ApiClient.ets`
- `entry/src/main/ets/services/StepCounterService.ets`

**具体问题**：
- Token可能过期或无效时没有正确处理
- 同步失败时没有重试机制
- 网络状态检查不完善

### 3. 字段命名不一致
**问题**：前端使用复数形式（`totalSteps`, `remainingSteps`），后端使用单数形式（`totalStep`, `remainingStep`）

## 🔧 修复方案

### 1. 修复后端步数计算逻辑

**需要修改的文件**：
`hssp-backend/hssp-service/hssp-user/src/main/java/com/hssp/service/service/impl/UserServiceImpl.java`

**修复代码**：
```java
// 修复后的逻辑：
if (recordDate.equals(LocalDate.now())) {
    // 获取今天之前的步数
    Integer previousTodaySteps = existingRecord != null ? existingRecord.getStepCount() : 0;
    
    // 只处理步数增加的情况，防止回退
    if (todaySteps > previousTodaySteps) {
        int stepDiff = todaySteps - previousTodaySteps;
        
        // 更新用户的今日步数
        user.setRemainingStep(todaySteps);
        
        // 总步数只增不减
        Integer oldTotalStep = user.getTotalStep() != null ? user.getTotalStep() : 0;
        int newTotalStep = oldTotalStep + stepDiff;
        user.setTotalStep(newTotalStep);
        
        log.info("今日步数增加 - 用户: {}, 之前: {}, 现在: {}, 差值: {}, 新总步数: {}", 
            userId, previousTodaySteps, todaySteps, stepDiff, newTotalStep);
    } else {
        // 步数减少，只更新今日步数，不减少总步数
        user.setRemainingStep(todaySteps);
        log.info("今日步数减少，仅更新今日步数 - 用户: {}, 之前: {}, 现在: {}", 
            userId, previousTodaySteps, todaySteps);
    }
} else {
    // 历史数据，只更新总步数（如果大于当前值）
    Integer oldTotalStep = user.getTotalStep() != null ? user.getTotalStep() : 0;
    if (todaySteps > 0) {
        int newTotalStep = oldTotalStep + todaySteps;
        user.setTotalStep(newTotalStep);
        
        log.info("历史步数更新 - 用户: {}, 日期: {}, 步数: {}, 旧总步数: {}, 新总步数: {}", 
            userId, recordDate, todaySteps, oldTotalStep, newTotalStep);
    }
}
```

### 2. 增强前端同步稳定性

**需要修改的文件**：
`entry/src/main/ets/services/StepCounterService.ets`

**修复内容**：
1. 增加Token有效性检查
2. 完善网络状态检测
3. 增加重试机制
4. 优化错误处理

```typescript
// 增强的同步逻辑
private async syncStepsToBackend(): Promise<void> {
    if (this.isSyncing) {
        console.log('步数同步正在进行中，跳过本次同步')
        return
    }

    if (this.lastStepCount <= 0) {
        console.log('步数为0，无需同步')
        return
    }

    // 检查网络连接
    if (!await this.checkNetworkConnection()) {
        console.log('网络连接不可用，跳过同步')
        return
    }

    // 检查Token有效性
    if (!await this.validateToken()) {
        console.log('Token无效，需要重新登录')
        return
    }

    const currentTime = Date.now()
    if (currentTime - this.lastSyncTime < 30000) {
        console.log('距离上次同步时间太短，跳过本次同步')
        return
    }

    try {
        this.isSyncing = true
        this.lastSyncTime = currentTime

        console.log(`开始同步步数到后端: ${this.lastStepCount}步`)
        
        // 检查用户是否已登录
        if (!StateStore.isLoggedIn) {
            console.log('用户未登录，跳过步数同步')
            return
        }

        // 获取当前日期
        const today = new Date().toISOString().split('T')[0]
        
        // 调用API上传步数（带重试）
        await this.retryApiCall(() => ApiService.uploadSteps(this.lastStepCount, today), 3)
        
        console.log('步数同步成功')
        
        // 更新本地存储
        StateStore.todaySteps = this.lastStepCount
        AppStorage.setOrCreate('todaySteps', this.lastStepCount)
        
    } catch (error) {
        const errorMessage = error instanceof Error ? error.message : String(error)
        console.error(`步数同步失败: ${errorMessage}`)
        
        // 记录失败日志，但继续运行
    } finally {
        this.isSyncing = false
    }
}
```

### 3. 统一字段命名规范

**解决方案**：
1. **建议方案**：修改后端实体类字段名为复数形式
2. **兼容方案**：在前端API客户端中保持字段转换

**推荐使用兼容方案**（不破坏现有数据库结构）：

在 `ApiClient.ets` 中保持现有的字段转换逻辑：
```typescript
// 转换后端字段为前端字段
const userInfo: UserInfo = {

```

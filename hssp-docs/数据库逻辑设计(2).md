## 数据库逻辑设计

### 1. 用户表 (`user`)

存储基础用户信息，作为各功能关联的主体。

| **字段名**        | **类型**       | **说明**             | **备注**              |
| ----------------- | -------------- | -------------------- | --------------------- |
| `id`              | `BIGINT`       | 主键 ID              | 分布式 ID 或自增      |
| `username`        | `VARCHAR(50)`  | 用户名/账号          | 唯一索引              |
| `password`        | `VARCHAR(50)`  | 密码                 | 用户密码(默认123456)  |
| `email`           | `VARCHAR(50)`  | 邮箱地址             | 登录凭证              |
| `nickname`        | `VARCHAR(50)`  | 昵称                 | 用于排行榜展示        |
| `sex`             | `SMALLINT`     | 性别                 | 1：男    2：女        |
| `avatar`          | `VARCHAR(255)` | 头像 URL             | 用于排行榜展示        |
| `total_points`    | `INT`          | 当前总积分           | 默认为 0              |
| `total_step`      | `INT`          | 总步数               | 默认为0               |
| `remaining_step`  | `INT`          | 剩余步数             | 默认为0               |
| `max_daily_steps` | `INT`          | 历史个人单日最高步数 | 默认为0               |
| `is_deleted`      | `BOOLEAN`      | 逻辑删除             | 0：未被删除 1：已删除 |
| `create_time`     | `DATETIME`     | 注册时间             |                       |
| `update_time`     | `DATETIME`     | 修改时间             |                       |

### 2. 管理员表 (`admin`)

存储管理员信息，作为各功能关联的主体。

| **字段名**    | **类型**       | **说明**    | **备注**             |
| ------------- | -------------- | ----------- | -------------------- |
| `id`          | `BIGINT`       | 主键 ID     | 分布式 ID 或自增     |
| `username`    | `VARCHAR(50)`  | 用户名/账号 | 唯一索引             |
| `password`    | `VARCHAR(50)`  | 密码        | 用户密码(默认123456) |
| `email`       | `VARCHAR(50)`  | 邮箱地址    | 登录凭证             |
| `nickname`    | `VARCHAR(50)`  | 昵称        | 用于排行榜展示       |
| `sex`         | `SMALLINT`     | 性别        | 1：男    2：女       |
| `avatar`      | `VARCHAR(255)` | 头像 URL    | 用于排行榜展示       |
| `create_time` | `DATETIME`     | 注册时间    |                      |
| `update_time` | `DATETIME`     | 修改时间    |                      |

### 3. 每日步数记录表 (`step_record`)

存储鸿蒙端上传的原始步数。为了方便查询“个人趋势”和“排行榜”，建议采用**日聚合**存储。

| **字段名**         | **类型**   | **说明**     | **备注**                   |
| ------------------ | ---------- | ------------ | -------------------------- |
| `id`               | `BIGINT`   | 主键 ID      |                            |
| `user_id`          | `BIGINT`   | 用户 ID      | 关联 user.id，建立索引     |
| `record_date`      | `DATE`     | 记录日期     | 格式：YYYY-MM-DD，联合索引 |
| `step_count`       | `INT`      | 今日步数累计 | 鸿蒙上传后修改(默认为0)    |
| `last_upload_time` | `DATETIME` | 最后同步时间 | 用于判断数据新鲜度         |

### 4. 积分兑换规则表 (`points_rule`)

管理员配置，用于前端 Vue 后台管理.

| **字段名**       | **类型**      | **说明** | **备注**       |
| ---------------- | ------------- | -------- | -------------- |
| `id`             | `INT`         | 规则 ID  |                |
| `rule_name`      | `VARCHAR(50)` | 规则名称 | 如“每千步兑换” |
| `steps_required` | `INT`         | 所需步数 | 例如：1000     |
| `points_awarded` | `INT`         | 获得积分 | 例如：10       |
| `is_active`      | `TINYINT`     | 是否启用 | 1-启用，0-禁用 |

### 5. 积分兑换日志表 (`points_log`)

记录兑换历史，用于鸿蒙端查看记录，并防止重复兑换。

| **字段名**      | **类型**   | **说明**       | **备注** |
| --------------- | ---------- | -------------- | -------- |
| `id`            | `BIGINT`   | 主键 ID        |          |
| `user_id`       | `BIGINT`   | 用户 ID        |          |
| `before_step`   | `INT`      | 兑换前的步数   |          |
| `after_step`    | `INT`      | 兑换后的步数   |          |
| `points_amount` | `INT`      | 兑换获得的积分 |          |
| `create_time`   | `DATETIME` | 兑换时间       |          |

### 6.  往期排行榜汇总表 (`step_rank_history`)

该表用于存储已经结束的周期（如：昨日、上周、上月）的最终排名结果。

| **字段名**     | **类型**      | **说明**     | **备注**                       |
| -------------- | ------------- | ------------ | ------------------------------ |
| `id`           | `BIGINT`      | 主键 ID      |                                |
| `user_id`      | `BIGINT`      | 用户 ID      | 关联 `user.id`                 |
| `period_type`  | `TINYINT`     | 周期类型     | 1:日排行，2:周排行，3:月排行   |
| `period_ident` | `VARCHAR(20)` | 周期标识     | 如：`2026-03-25` 或 `2026-W12` |
| `total_steps`  | `INT`         | 周期内总步数 | 该周期结算时的累计步数         |
| `rank_num`     | `INT`         | 最终名次     | 结算后的排名位置               |
| `settle_time`  | `DATETIME`    | 结算时间     | 统计任务执行的时间             |

# 步数上传和排行榜功能测试指南

## 功能概述

本次更新完成了以下两个核心功能：

### 1. 步数上传功能
- **接口**: `POST /user/steps/upload`
- **功能**: 用户上传每日步数，系统自动保存到数据库并同步更新Redis排行榜
- **特性**:
  - 支持指定日期或默认今天
  - 自动去重（同一用户同一天只保留最大步数）
  - 实时更新用户总步数和剩余步数
  - 同步更新日榜、周榜、月榜三个Redis ZSet

### 2. 排行榜功能
- **接口**: `GET /user/rankings?periodType=day&pageNum=1&pageSize=10`
- **功能**: 查询不同周期的步数排行榜
- **参数**:
  - `periodType`: day(日榜) / week(周榜) / month(月榜)
  - `pageNum`: 页码，默认1
  - `pageSize`: 每页数量，默认10
- **返回**: PageResult分页数据结构

## 常见问题排查

### 1. 步数上传失败
- 检查Token是否正确传递
- 检查JWT_SWITCH配置（开发时可设为0关闭验证）
- 查看后端日志，确认错误信息

### 2. 排行榜数据为空
- 确认Redis服务正常运行
- 检查Redis中是否有数据：`KEYS hssp:rank:*`
- 确认步数上传时Redis更新成功（查看日志）

### 3. 前端显示异常
- 检查网络请求是否成功（DevEco Studio控制台）
- 验证API响应格式是否符合预期
- 清除应用缓存后重试

### 4. 数据库连接问题
- 检查 `application.yml` 中的数据源配置
- 确认MySQL服务已启动
- 验证数据库名、用户名、密码是否正确

## 技术实现要点

### 后端实现
1. **StepRecord实体**: 新增步数记录实体类，映射step_record表
2. **uploadSteps方法**: 
   - 事务控制保证数据一致性
   - 自动判断日期，支持历史数据补录
   - 同一日期取最大步数，避免重复累加
   - 同步更新Redis三个排行榜ZSet
3. **Redis排行榜**:
   - 日榜: `hssp:rank:daily:{yyyyMMdd}` - 直接set分数
   - 周榜: `hssp:rank:week:{yyyyww}` - incrementScore累加
   - 月榜: `hssp:rank:month:{yyyyMM}` - incrementScore累加

### 前端实现
1. **RankingPage优化**:
   - 正确解析PageResult分页结构
   - 处理后端RankVO字段映射
   - 添加空数据和加载状态处理
2. **数据类型转换**:
   - totalSteps从Double转为Integer显示
   - 处理可能为空的字段（nickname、avatar等）

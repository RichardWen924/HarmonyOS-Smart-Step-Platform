# 步数同步与积分兑换功能说明

## 功能概述

本项目实现了完整的步数同步和积分兑换功能，包括：

1. **步数实时监控与同步** - 自动监控设备步数变化并同步到后端数据库和Redis
2. **积分兑换系统** - 支持步数兑换积分，按最优规则组合计算
3. **兑换预览功能** - 在兑换前预览可获得积分和剩余步数
4. **数据持久化** - 所有数据同步到MySQL数据库和Redis缓存

## 核心组件

### 1. StepCounterService (步数监控服务)

**位置**: `entry/src/main/ets/services/StepCounterService.ets`

**功能**:
- 监控设备步数传感器变化
- 自动同步步数到后端（每分钟同步一次，或步数变化超过100步时立即同步）
- 支持手动同步和测试功能

**关键特性**:
- 智能同步策略，避免频繁请求
- 用户登录状态检查
- 错误处理和重试机制
- 本地数据缓存

### 2. ApiService (API服务层)

**位置**: `entry/src/main/ets/services/ApiService.ets`

**功能**:
- 提供统一的API调用接口
- 错误处理和类型转换
- 数据验证和业务逻辑封装

**主要API**:
- `uploadSteps(steps, date)` - 上传步数到数据库和Redis
- `previewExchange(steps)` - 预览兑换结果
- `exchangePoints(steps)` - 执行积分兑换
- `getPointRules()` - 获取兑换规则
- `getUserInfo()` - 获取用户信息

### 3. PointsExchangePage (积分兑换页面)

**位置**: `entry/src/main/ets/pages/PointsExchangePage.ets`

**功能**:
- 显示用户当前步数和积分
- 提供步数输入和快捷选择
- 兑换预览和确认功能
- 实时更新本地数据

**UI特性**:
- 响应式设计，适配不同屏幕
- 直观的步数和积分显示
- 友好的用户交互体验
- 错误提示和加载状态

## 数据流设计

### 步数同步流程

```
设备步数传感器 → StepCounterService → ApiService → 后端API
    ↓
本地存储更新 ← 同步成功 ← 数据库/Redis更新
```

### 积分兑换流程

```
用户输入步数 → 预览兑换 → 确认兑换 → 执行兑换
    ↓              ↓           ↓           ↓
规则验证 → 计算可得积分 → 用户确认 → 更新数据库
```

## 后端API接口

### 步数上传接口
- **URL**: `POST /api/steps/upload`
- **参数**: `{steps: number, date: string}`
- **功能**: 将步数同步到MySQL数据库和Redis缓存

### 积分兑换预览接口
- **URL**: `POST /api/points/preview`
- **参数**: `{steps: number}`
- **功能**: 计算按最优规则组合可获得的积分

### 积分兑换接口
- **URL**: `POST /api/points/exchange`
- **参数**: `{steps: number}`
- **功能**: 执行积分兑换，更新用户步数和积分

## 数据库设计

### 用户步数表 (user_steps)
```sql
CREATE TABLE user_steps (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    steps INT NOT NULL,
    date DATE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY unique_user_date (user_id, date)
);
```

### 积分兑换规则表 (point_rules)
```sql
CREATE TABLE point_rules (
    id INT PRIMARY KEY AUTO_INCREMENT,
    steps_required INT NOT NULL,
    points_awarded INT NOT NULL,
    rule_name VARCHAR(100),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Redis缓存设计

### 用户今日步数缓存
```
key: user:steps:{userId}:{date}
value: 步数值
过期时间: 24小时
```

### 用户积分缓存
```
key: user:points:{userId}
value: 积分值
过期时间: 1小时
```
